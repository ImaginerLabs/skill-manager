## 填写示例

````markdown
## 📋 修复意图分析

### 基本信息

- **Commit**：`a1b2c3d4` - fix: 修复购物车数量为空时的崩溃问题
- **作者**：张三
- **时间**：2026-03-15 14:30
- **改动文件数**：2 个文件

---

### 问题根因

> 这个修复解决了什么问题？

当用户购物车中的商品数量字段从后端返回 `null` 时，前端直接对 `null` 进行数学运算，导致页面崩溃（NaN 显示问题）。

---

### 改动范围

| 文件                          | 改动类型 | 改动目的                         |
| ----------------------------- | -------- | -------------------------------- |
| `src/pages/cart/CartItem.tsx` | 修改     | 对 quantity 字段增加空值兜底处理 |
| `src/utils/cartHelper.ts`     | 修改     | 计算总价时增加 NaN 过滤          |

---

### 核心逻辑变化

**改动前的行为**：
直接使用 `item.quantity` 进行计算，当值为 `null` 时产生 NaN。

**改动后的行为**：
使用 `item.quantity ?? 0` 兜底，确保计算结果始终为有效数字。

**关键代码变化**（原始分支）：
\```diff

- const total = item.quantity \* item.price;

* const total = (item.quantity ?? 0) \* item.price;
  \```

## 注意事项

- **风险等级定义**：
  - **高**：当前分支可能没有对应的实现基础，需要大量重写
  - **中**：结构有差异，但核心逻辑可以复用
  - **低**：差异较小，可以直接应用或稍作调整

- **向用户确认的时机**：
  - 分析完成后，输出此报告，等待用户确认再动手
  - 如果风险等级有"高"，必须先与用户讨论方案
