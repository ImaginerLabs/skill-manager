## 一、需求概述

### 需求类型

> Bug 修复

### 需求目标

> 修复当后端返回的购物车商品数量字段为 `null` 时，前端页面崩溃（显示 NaN）的问题。

### 背景说明

> 后端接口在某些异常情况下会返回 `quantity: null`，前端未做空值处理，导致计算总价时出现 NaN，页面显示异常。

### 改动文件清单

| 文件绝对路径                                          | 改动类型 | 改动说明                         |

---

## 三、核心逻辑说明

### 改动前的行为

直接使用 `item.quantity` 进行乘法运算，当值为 `null` 时产生 NaN，导致总价显示异常。

### 改动后的行为

使用 `item.quantity ?? 0` 兜底，确保计算结果始终为有效数字，页面正常显示。

### 关键代码变化

**文件：`/Users/zhangsan/project/src/utils/cartHelper.ts`**

改动前：
\```typescript
export function calcTotal(item: CartItem): number {
return item.quantity \* item.price;
}
\```

改动后：
\```typescript
export function calcTotal(item: CartItem): number {
// 兜底处理：quantity 可能为 null（后端异常情况）
return (item.quantity ?? 0) \* item.price;
}
\```

### 步骤 1：修复总价计算函数

**操作目标**：确保计算总价时，数量字段为 null 不会产生 NaN

**操作文件**：找到项目中计算购物车总价的工具函数，使用绝对路径定位，例如 `/Users/xxx/your-project/src/utils/cartHelper.ts`（路径因项目而异）

**具体操作**：
找到类似 `calcTotal` 或 `getTotalPrice` 的函数，在数量字段参与计算前增加空值兜底：

\```typescript
// 修改前

// 修改后（根据项目风格选择其中一种）
const total = (item.quantity ?? 0) _ item.price;
// 或
const total = (item.quantity || 0) _ item.price;
\```

**验证方式**：在浏览器控制台执行 `calcTotal({ quantity: null, price: 100 })`，结果应为 `0` 而非 `NaN`。

## 六、注意事项

### ⚠️ 常见问题

**问题 1：使用 `|| 0` 还是 `?? 0`？**

> 原因：`|| 0` 会将 `0` 也替换为 `0`（无影响），但会将空字符串 `""` 也替换；`?? 0` 只处理 `null` 和 `undefined`，更精确。
> 解决：推荐使用 `?? 0`，如果项目不支持可选链语法，使用 `|| 0` 也可接受。

> `CartItem` 类型定义不要直接复制，应使用接收方项目自己的类型定义。
