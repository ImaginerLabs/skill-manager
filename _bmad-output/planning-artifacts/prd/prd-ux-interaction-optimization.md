# PRD 增补提案 — UX 交互优化

**Author:** Alex / Sally (UX Designer)
**Date:** 2026-04-15
**类型:** 现有产品增补 PRD（基于代码实现与 UX 规范差距分析）
**依赖:** Epic 0 ~ Epic 8 全部完成

---

## Executive Summary

本提案基于对 Skill Manager 代码实现与 UX 设计规范的深度对比分析，识别出五个高价值交互优化方向。这些优化不是功能缺失，而是**已有功能在交互层面的体验落差**——预览面板响应式行为缺失、搜索高亮未实现、键盘导航体系不完整、同步流程缺乏渐进引导、分类切换上下文断裂。

**核心洞察：** 产品功能已全部交付（V2 功能全量完成），但 UX 规范中定义的多项交互模式在代码实现中被遗漏或简化，导致用户在日常使用中产生可感知的体验摩擦。本提案的目标是**弥合规范与实现之间的差距**，让每一个已有功能的交互体验达到设计规范定义的标准。

**预计规模：** 5 个 Story，1 名前端开发者，预计 2 周完成。

---

## 痛点清单（按优先级排序）

### P0 — 影响最广的体验问题

#### 痛点 1：预览面板响应式行为缺失

**现象：** 当前 `AppLayout.tsx` 中，预览面板在 Skill 浏览页始终显示（`showPreview = isSkillBrowsePage`），且宽度固定。Standard 断点（1024-1439px）下内容被严重挤压，用户无法主动关闭预览面板获得更大浏览空间。

**根因：** `previewOpen` 状态未使用，UX 规范在 Standard 断点下定义的"预览激活/未激活两种状态"完全缺失。

**用户影响：** Standard 断点用户被迫接受常驻预览面板，主内容区宽度不足，卡片网格只能展示 2 列。

---

#### 痛点 2：搜索结果无关键词高亮，Command Palette 与页面搜索割裂

**现象：** `SkillCard.tsx` 直接渲染 `skill.name` 和 `skill.description`，搜索匹配的关键词没有视觉标记。Command Palette 搜索 Skill 后调用 `selectSkill(skill.id)` 并 `navigate("/")`，但不将搜索词传递给页面搜索框。

**根因：** 搜索高亮组件未实现；Command Palette 选择结果后未同步页面搜索状态。

**用户影响：** 用户无法快速判断"为什么这条结果出现在搜索结果中"；从 ⌘K 选中 Skill 后跳转到页面看到的是未过滤的全量列表，搜索上下文断裂。

---

### P1 — 核心效率提升

#### 痛点 3：键盘导航体系不完整

**现象：** UX 规范定义了 J/K 上下导航、右键上下文菜单、Space 预览选中 Skill 等键盘快捷键，但代码中仅注册了 `⌘B` 和 `Space`（且 Space 语义与规范不一致），J/K 导航和右键菜单完全缺失。

**根因：** 键盘快捷键注册不完整，SkillCard 缺少 `onContextMenu` 处理。

**用户影响：** 键盘用户无法实现"不碰鼠标完成全部操作"，与 IDE 级别的键盘操作效率预期不符。

---

#### 痛点 4：同步流程缺乏渐进引导

**现象：** 同步按钮下方仅显示纯文本摘要，用户无法在点击前了解"哪些 Skill 将同步到哪里"；Diff 功能隐藏在 SplitButton 下拉中；失败项没有"重试"入口；同步过程中无进度指示。

**根因：** 同步交互设计偏"功能可用"而非"体验流畅"，缺少操作前确认、进度反馈和错误恢复机制。

**用户影响：** 同步操作是"盲操作"，用户在同步前无法确认操作范围，失败后只能全部重新同步。

---

### P2 — 体验打磨

#### 痛点 5：分类/来源切换的上下文断裂

**现象：** `SecondarySidebar.tsx` 切换分类/来源视图时，列表内容硬切换产生闪烁；切换后选中 Skill 瞬间跳变无过渡；页面上无面包屑/筛选状态指示。

**根因：** 列表切换缺少过渡动效；筛选状态缺少可视化指示。

**用户影响：** 用户在分类导航中容易迷失上下文，不知道"当前在哪里"和"如何返回"。

## 功能需求（新增 FR）

### 预览面板智能推拉式

- **FR-UX-01：** Wide（≥1440px）断点下，三栏常驻布局，预览面板固定右侧 400px
- **FR-UX-02：** Standard（1024-1439px）断点下，预览面板默认隐藏，点击 Skill 卡片后以 push 模式从右侧推入（宽度 360px），主内容区自动收缩；再次点击同一卡片或按 Escape 收起
- **FR-UX-03：** Compact（<1024px）断点下，预览面板以 overlay 模式从右侧滑出覆盖主内容区
- **FR-UX-04：** 预览面板顶部添加关闭按钮（×），并支持 `⌘\` 快捷键切换预览
- **FR-UX-05：** 预览面板展开/收起应有平滑过渡动画（transform + opacity，200ms ease-in-out）

### 搜索体验增强

- **FR-UX-06：** 创建 `<HighlightText>` 组件，对 SkillCard 的名称和描述中匹配搜索关键词的片段使用 `<mark>` 标签高亮（绿色背景 `bg-green-500/20 text-green-400`）
- **FR-UX-07：** Command Palette 选中 Skill 后，同步设置页面搜索框的 `searchQuery` 状态，确保页面展示上下文一致
- **FR-UX-08：** 搜索框旁显示实时匹配计数（如 `12/35`），并通过 `aria-live="polite"` 播报给屏幕阅读器

### 键盘导航体系补全

- **FR-UX-09：** 在 Skill 网格/列表中实现 J/K 导航，J 键聚焦下一张卡片，K 键聚焦上一张卡片，配合 `scrollIntoView({ block: 'nearest' })` 确保可见
- **FR-UX-10：** 为 SkillCard 添加右键上下文菜单（使用 shadcn/ui `ContextMenu`），包含"编辑元数据 / 同步到 IDE / 复制路径 / 删除"四个操作项
- **FR-UX-11：** 修正 Space 键语义：从"切换预览面板"改为"预览选中 Skill"（选中并展示预览），移除预览面板的 Space 切换
- **FR-UX-12：** Skill 卡片获得焦点时，按 Delete/Backspace 弹出确认删除对话框

### 同步流程渐进引导

- **FR-UX-13：** 点击同步前，在按钮下方展开轻量摘要面板，列出"将要同步的 Skill 列表 + 目标路径"，用户确认后再执行
- **FR-UX-14：** 将 Diff 预览按钮从 SplitButton 下拉中独立出来，作为与同步按钮平级的"预览变更"按钮；多目标时提示用户选择对比目标
- **FR-UX-15：** 同步结果列表的 failed 项旁添加"重试"按钮，点击后仅重新同步该 Skill
- **FR-UX-16：** 同步过程中展示进度条 + 文字（如 `12/35 Skill 已同步`），而非仅禁用按钮

### 分类切换上下文保持

- **FR-UX-17：** 在主内容区顶部工具栏下方添加筛选路径面包屑，如 `全部 > coding` 或 `来源: codebuddy-hub`，点击面包屑中的层级可快速回退
- **FR-UX-18：** 分类切换时 Skill 列表使用 `AnimatePresence` + `motion.div layout` 动画，卡片位置平滑过渡
- **FR-UX-19：** 切换 Skill 时预览面板元数据区域淡入淡出（150ms），而非内容瞬间替换
- **FR-UX-20：** 面包屑右侧提供 × 按钮快速清除筛选

---

## 非功能需求

- **NFR-UX-01：** 预览面板 push/overlay 切换动画帧率 ≥ 60fps，不得阻塞主线程渲染
- **NFR-UX-02：** 搜索高亮渲染不得影响 Skill 列表加载性能（< 200ms，500 Skill 规模）
- **NFR-UX-03：** 所有新增动画遵循 `prefers-reduced-motion` 媒体查询，保障无障碍访问
- **NFR-UX-04：** 右键菜单的键盘操作遵循 WAI-ARIA Menu Pattern（方向键导航、Escape 关闭）
- **NFR-UX-05：** 同步进度条更新频率不低于每 500ms 一次，确保用户感知到进展

## 用户旅程（改进后）

### 旅程 A：Standard 断点用户浏览 Skill（改进后）

1. 用户打开 Skill Manager，主内容区占据大部分空间，卡片网格展示 3-4 列
2. 用户点击某个 Skill 卡片 → 预览面板以 push 模式从右侧平滑推入，主内容区自动收缩
3. 用户阅读预览后按 Escape → 预览面板平滑收起，主内容区恢复原宽度

### 旅程 B：用户搜索并定位 Skill（改进后）

1. 用户在搜索框输入"review"，实时看到匹配计数 `3/35`
2. SkillCard 中"review"关键词以绿色高亮显示，0.5 秒内识别匹配位置
3. 用户按 ⌘K 打开 Command Palette → 输入"review" → 选中 Skill → 页面同步展示搜索结果

### 旅程 C：键盘用户高效操作（改进后）

1. 用户按 J/K 在 Skill 卡片间导航，当前聚焦卡片有明显视觉指示
2. 按 Space 预览选中 Skill → 按 Escape 关闭预览
3. 在 Skill 卡片上按右键 → 上下文菜单弹出"编辑元数据 / 同步 / 复制路径 / 删除"
4. 按 Delete 弹出确认删除对话框

### 旅程 D：用户执行同步（改进后）

1. 用户勾选 Skill → 点击"预览变更"查看 Diff → 确认无误
2. 点击"开始同步" → 摘要面板展开展示操作范围 → 确认执行
3. 进度条实时推进 `12/35 Skill 已同步` → 2 项失败
4. 用户点击失败项旁的"重试"按钮 → 仅重新同步失败的 Skill

## 成功标准

| 指标 | 当前状态 | 目标状态 | 测量方式 |
|------|----------|----------|----------|
| Standard 断点卡片列数 | 2 列 | 3-4 列（预览隐藏时） | 视觉验证 |
| 搜索关键词识别时间 | 需逐行阅读 | < 0.5 秒（高亮标记） | 用户测试 |
| Command Palette 到页面搜索连贯性 | 断裂（搜索词丢失） | 连贯（搜索词同步） | E2E 测试 |
| 键盘可完成操作覆盖率 | ~40% | ~95% | 操作路径分析 |
| 同步操作前确认率 | 0%（盲操作） | 100%（摘要确认） | E2E 测试 |
| 同步失败可重试 | 否（需全部重新同步） | 是（单项重试） | 功能验证 |
| 分类切换感知流畅度 | 硬切换闪烁 | 平滑过渡 | 视觉验证 |

## 风险与约束

| 风险 | 严重度 | 缓解策略 |
|------|--------|----------|
| 预览面板 push 模式可能影响现有布局计算 | 🟡 中 | 先实现 overlay 模式作为降级方案，push 模式逐步适配 |
| HighlightText 组件需处理多关键词 AND 匹配的高亮逻辑 | 🟢 低 | 使用正则全局替换，先支持单关键词高亮 |
| Framer Motion 引入可能增加包体积 | 🟡 中 | 项目已有 `@dnd-kit`，评估是否可用 CSS transition 替代 layout 动画 |
| 右键菜单需与 SkillCard 的 button 元素交互兼容 | 🟢 低 | 使用 shadcn/ui ContextMenu 原生支持 |
